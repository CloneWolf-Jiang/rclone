📋 BUG List Overview

BUG #1: Premature EOF Causing Infinite Retries PENDING FIX

Problem Summary

After backend object deletion, client Seek operations trigger frequent EOF errors. VFS cache retains original object size in memory. ReadAt returns EOF at position 0, which is less than cached fh.size, preventing break condition from triggering. Results in 10+ error log lines per second.

Current Logic

File: vfs/read.go lines 306-313, ReadFileHandle.readAt()

if err == io.EOF { if newOffset == fh.size || fh.sizeUnknown { break // Normal EOF: reached expected position } // Condition not met → infinite retry loop }

Root Cause Analysis

The variable fh.size is an in-memory cached value of the original object size. When backend object becomes inaccessible, disk read returns EOF at newOffset=0, which is far less than cached fh.size. The break condition never becomes true, causing the loop to persist indefinitely.

Trigger Scenarios

  • Jellyfin/Plex client performing Seek operations on mounted files
  • Backend object becomes inaccessible (deleted, path changed, etc.)
  • VFS cache not immediately updated due to poll-interval delays

Proposed Fix

Add premature EOF detection in vfs/read.go:

if err == io.EOF { if newOffset == fh.size || fh.sizeUnknown { break // Normal: reached expected position } else if newOffset > 0 && newOffset < fh.size { // Premature EOF: object became unavailable break // Immediate break from retry loop } }

Actual Change Status

Not yet implemented on main branch (pending verification of libfuse 3.18.1 upgrade effectiveness)

BUG #2: Upload After Parent Directory Deletion FIXED

Problem Summary

After backend directory deletion, VFS cache files still attempt upload. Upload(.partial) succeeds (HTTP 200) but atomic rename fails ("object not found"), causing infinite retry loops. Cache items remain marked as Dirty, unable to be evicted.

Current Logic

File: vfs/vfscache/item.go lines 590-620, _store() method

// Direct upload attempt without parent dir validation cacheObj, err := item.c.fcache.NewObject(ctx, item.name) // ... o, err = operations.Copy(ctx, item.c.fremote, o, name, cacheObj) // Copy succeeds but Move(.partial → final) fails

Root Cause Analysis

VFS cache layer assumes parent directory always exists. After backend directory deletion:

  1. PUT upload succeeds (creates temporary file): HTTP 200
  2. Move operation fails: Backend returns "object not found" (parent dir is gone)
  3. VFS marks item as Dirty, schedules 5-minute retry
  4. File never becomes clean, cache entries accumulate indefinitely

Trigger Scenarios

Key Scenario: Backend directory deleted → Jellyfin metadata updated → VFS marks file as Dirty → Upload attempt fails

Log sequence:
FindLeaf: "Season05" not found in "/TV/Series"
Upload: HTTP 200 ✓
Move: failed to get src object: object not found ✗

Proposed Fix (Plan A)

In vfs/vfscache/item.go _store() method, validate parent directory existence before operations.Copy():

// Validate parent directory exists to prevent invalid uploads parentDir := path.Dir(item.name) if parentDir != "" && parentDir != "." { _, dirErr := item.c.fremote.List(ctx, parentDir) if dirErr == fs.ErrorDirNotFound { // Parent dir missing → mark clean → return success item.info.Dirty = false item._save() return nil } else if dirErr != nil { return fmt.Errorf("check parent directory failed: %w", dirErr) } } // Parent dir exists → proceed with operations.Copy()

Code Changes

Before (original Fork - lines 590-620):

if cacheObj != nil { o, name := item.o, item.name unlockMutexForCall(&item.mu, func() { o, err = operations.Copy(ctx, item.c.fremote, o, name, cacheObj) }) if err != nil { if errors.Is(err, fs.ErrorCantUploadEmptyFiles) { fs.Errorf(name, "Writeback failed: %v", err) return nil } return fmt.Errorf("vfs cache: failed to transfer file: %w", err) } }

After (final version - lines 600-634):

if cacheObj != nil { // Check if the parent directory exists on the remote before uploading parentDir := path.Dir(item.name) if parentDir != "" && parentDir != "." { _, dirErr := item.c.fremote.List(ctx, parentDir) if dirErr == fs.ErrorDirNotFound { fs.Infof(item.name, "vfs cache: skipping upload - parent directory '%s' not found on backend", parentDir) item.info.Dirty = false err = item._save() if err != nil { fs.Errorf(item.name, "vfs cache: failed to write metadata file: %v", err) } return nil } else if dirErr != nil { return fmt.Errorf("vfs cache: failed to check parent directory: %w", dirErr) } } o, name := item.o, item.name unlockMutexForCall(&item.mu, func() { o, err = operations.Copy(ctx, item.c.fremote, o, name, cacheObj) }) if err != nil { if errors.Is(err, fs.ErrorCantUploadEmptyFiles) { fs.Errorf(name, "Writeback failed: %v", err) return nil } return fmt.Errorf("vfs cache: failed to transfer file from cache to remote: %w", err) } }

Actual Implementation

✅ IMPLEMENTED in vfs/vfscache/item.go

Changes made:

  • Added import "path"
  • Parent directory check before operations.Copy
  • If ErrorDirNotFound: set Dirty=false and return nil

Effect: Files no longer enter infinite upload retry loops. RemoveNotInUse() can finally evict cache entries.
