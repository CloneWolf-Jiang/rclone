📋 BUG 列表总览

BUG #1: Premature EOF 导致无限重试 待修复

问题简述

后端文件被删除后，客户端 Seek 操作触发频繁 EOF 错误。FUSE 缓存仍保留原文件大小，读取返回 EOF 但位置 < 预期大小，导致重试不中断，每秒产生 10+ 日志行。

现有逻辑

文件: vfs/read.go 行 306-313 的 ReadFileHandle.readAt()

if err == io.EOF { if newOffset == fh.size || fh.sizeUnknown { break // 正常 EOF } // 条件不满足 → 无限重试 }

导致 BUG 原因

预期大小 (fh.size) 是 VFS 缓存在内存中的旧值。当后端对象删除后，磁盘上读取返回 EOF，但 EOF 位置 (newOffset=0) 远小于缓存的 fh.size。break 条件永不成立，导致循环体持续执行。

触发场景

  • Jellyfin/Plex 客户端进行 Seek 操作
  • 后端对象由于任何原因不可访问（删除、路径变更等）
  • FUSE 缓存未及时更新（poll-interval 导致延迟）

建议修改

在 vfs/read.go 中添加 premature EOF 检测：

if err == io.EOF { if newOffset == fh.size || fh.sizeUnknown { break // 正常：已读到预期位置 } else if newOffset > 0 && newOffset < fh.size { // Premature EOF：文件中间出现 EOF // 可能原因：后端对象不可用、连接断开等 break // 立即中止重试 } }

实际修改

暂未在主分支实施（需等待 libfuse 3.18.1 升级效果验证）

BUG #2: 父目录删除后仍然上传 已修复

问题简述

后端目录被删除后，VFS 缓存中的文件仍尝试上传。上传(.partial)成功但重命名失败（"object not found"），导致无限重试。缓存项永远无法标记为干净，占用磁盘和内存。

现有逻辑

文件: vfs/vfscache/item.go 行 590-620 的 _store() 方法

// 直接尝试上传，不验证父目录 cacheObj, err := item.c.fcache.NewObject(ctx, item.name) // ... o, err = operations.Copy(ctx, item.c.fremote, o, name, cacheObj) // Copy 成功但 Move(.partial → final) 失败

导致 BUG 原因

VFS cache 层假设父目录始终存在。当后端目录被删除时：

  1. PUT 上传成功（创建临时文件）：HTTP 200
  2. Move 操作失败：Alist 返回 "object not found"（父目录消失）
  3. VFS 标记为脏 (Dirty=true)，计划 5 分钟后重试
  4. 文件永远无法标记为干净，缓存项累积

触发场景

关键场景: 后端目录被删除 → Jellyfin Metadata 文件修改 → VFS 标记为脏 → 尝试上传

日志序列：
FindLeaf: 在 "/TV/斗破苍穹" 中未找到 "Season05"
Upload: HTTP 200 ✓
Move: failed to get src object: object not found ✗

建议修改（Plan A）

在 vfs/vfscache/item.go 中的 _store() 方法执行 operations.Copy() 之前，先检查父目录是否存在：

// 校验父目录存在，避免无效上传 parentDir := path.Dir(item.name) if parentDir != "" && parentDir != "." { _, dirErr := item.c.fremote.List(ctx, parentDir) if dirErr == fs.ErrorDirNotFound { // 父目录不存在→标记为干净→返回成功 item.info.Dirty = false item._save() return nil } else if dirErr != nil { return fmt.Errorf("check parent directory failed: %w", dirErr) } } // 父目录存在→继续执行 operations.Copy()

代码变更

修改前（原始Fork - 行590-620）：

if cacheObj != nil { o, name := item.o, item.name unlockMutexForCall(&item.mu, func() { o, err = operations.Copy(ctx, item.c.fremote, o, name, cacheObj) }) if err != nil { if errors.Is(err, fs.ErrorCantUploadEmptyFiles) { fs.Errorf(name, "Writeback failed: %v", err) return nil } return fmt.Errorf("vfs cache: failed to transfer file: %w", err) } }

修改后（最终版本 - 行600-634）：

if cacheObj != nil { // 上传前检查父目录是否存在 parentDir := path.Dir(item.name) if parentDir != "" && parentDir != "." { _, dirErr := item.c.fremote.List(ctx, parentDir) if dirErr == fs.ErrorDirNotFound { fs.Infof(item.name, "vfs cache: skipping upload - parent directory '%s' not found on backend", parentDir) item.info.Dirty = false err = item._save() if err != nil { fs.Errorf(item.name, "vfs cache: failed to write metadata file: %v", err) } return nil } else if dirErr != nil { return fmt.Errorf("vfs cache: failed to check parent directory: %w", dirErr) } } o, name := item.o, item.name unlockMutexForCall(&item.mu, func() { o, err = operations.Copy(ctx, item.c.fremote, o, name, cacheObj) }) if err != nil { if errors.Is(err, fs.ErrorCantUploadEmptyFiles) { fs.Errorf(name, "Writeback failed: %v", err) return nil } return fmt.Errorf("vfs cache: failed to transfer file from cache to remote: %w", err) } }

实际修改

✅ 已在 vfs/vfscache/item.go 实施

修改内容：

  • 添加 import "path"
  • 在 _store() 的 operations.Copy 前添加父目录检查
  • 若 ErrorDirNotFound，标记 Dirty=false 并返回成功

效果: 文件不再进入无限上传重试循环，RemoveNotInUse() 能最终清理缓存
