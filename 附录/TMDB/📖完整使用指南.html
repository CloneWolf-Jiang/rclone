自定义规则完全指南

什么是自定义规则？

规则 = 正则表达式

规则用来识别和提取文件名中的季号和集号。

规则的结构 

{
  "name": "规则的名称（如：01x01格式）",
  "pattern": "正则表达式（如：(\\d{1,2})x(\\d{1,2})）",
  "description": "规则描述（可选，如：标准的##x##格式）",
  "enabled": true 或 false（是否启用）
}

正则表达式基础速成

关键符号速查

符号 含义 示例
() 捕获组（必须有 2 个！） (\d+)
\d 任意数字 [0-9] \d{1,2} 匹配 1-2 位数字
{1,2} 重复次数 1 到 2 次 \d{1,2}
\s 空白字符（空格、制表符等） \s* 匹配 0 个或多个空格
[...] 字符集 [a-zA-Z] 匹配字母
| 或者 S|s 匹配 S 或 s
. 任意字符（除换行） 通常需要转义 \.
^ 字符串开头 ^S01
$ 字符串末尾 E01$

必须记住：2 个捕获组 

✅ 正确：   (\d{1,2})x(\d{1,2})
           第 1 组(季)  第 2 组(集)

❌ 错误：   \d{1,2}x\d{1,2}        → 没有括号！
❌ 错误：   (\d{1,2})               → 只有 1 个括号！
❌ 错误：   (\d{1,2})-(\d{1,2})-(\d{1,2}) → 3 个括号！

常见文件格式和规则

格式 1：01x01 

文件名示例：Breaking Bad 01x01 Pilot.mkv
正则表达式：(\d{1,2})x(\d{1,2})
规则名称：01x01 格式

格式 2：S01E01 

文件名示例：Breaking Bad S01E01 Pilot.mkv
正则表达式：[Ss](\d{1,2})[Ee](\d{1,2})
规则名称：S##E##标准格式

格式 3：[01] 01（动画格式） 

文件名示例：[Anime] Title [01] 01 Title.mkv
正则表达式：[\[\(]\s*(\d{1,2})\s*[\]\)]\s*(\d{1,2})
规则名称：动画 [##] ## 格式
说明：支持 [01] 或 (01) 两种括号

格式 4：01.01 

文件名示例：Breaking Bad 01.01 Pilot.mkv
正则表达式：(\d{1,2})\.(\d{1,2})
规则名称：点号格式

格式 5：01-01 

文件名示例：Breaking Bad 01-01 Pilot.mkv
正则表达式：(\d{1,2})-(\d{1,2})
规则名称：连字号格式

  -H "Content-Type: application/json" \
  -d '{
    "name": "01x01格式",
    "pattern": "(\\d{1,2})x(\\d{1,2})",
    "description": "标准的 ##x## 格式"
  }'

响应：

{
  "success": true,
  "message": "规则已添加: 01x01格式"
}

获取所有规则 

curl http://localhost:8000/api/get-rules

响应：

{
  "rules": [
    {
      "name": "01x01格式",
      "pattern": "(\\d{1,2})x(\\d{1,2})",
      "description": "标准的 ##x## 格式",
      "enabled": true
    }
  ]
}

测试规则

  -H "Content-Type: application/json" \
  -d '{

失败响应：

curl -X POST http://localhost:8000/api/update-rule \
  -H "Content-Type: application/json" \

    "name": "01x01格式",

常见问题 & 故障排除

启动和配置

Q: 启动时报错 "ModuleNotFoundError"

A: 安装缺失的依赖库

pip install -r requirements-tmdb.txt

或手动安装：

pip install requests

Q: 访问 localhost:8000 打不开

A: 检查以下几点：

  1. 程序是否正确启动了？（看执行窗口有没有错误）
  2. 是否在输入地址时用了正确的端口？
  3. 防火墙是否阻止了连接？

Q: API 密钥无效

A: 检查：

  • 密钥是否从官方来源复制：https://www.themoviedb.org/settings/api
  • 是否有多余空格？
  • 网络连接是否正常？
  • 密钥是否过期？（在 TMDB 重新生成）

文件名识别

Q: 分析文件名时显示"格式未识别"

A: 有两个原因，按顺序排查：

原因 1：你的格式不在内置规则中

解决：创建自定义规则
  1. 在"自定义正则表达式"框输入你的格式对应的正则
  2. 点"测试正则"验证
  3. 点"保存为规则"永久保存
  详见：【自定义规则完全指南】

原因 2：你的规则被禁用了

解决：启用规则
  1. 查看左侧"自定义规则"列表
  2. 找到对应的规则
  3. 勾选 ☑️ 启用它

Q: 正则表达式测试失败

A: 99% 的原因是这 3 个问题：

问题 1：缺少捕获组 ()

❌ 错误：\d{1,2}x\d{1,2}
✅ 正确：(\d{1,2})x(\d{1,2})

问题 2：特殊符号未转义

❌ 错误：(\d.2)x(\d.2)        点号没转义！
✅ 正确：(\d{1,2})x(\d{1,2})

问题 3：重复范围错误

❌ 错误：(\d{1-2})x(\d{1-2})   用了 - 代替 ,
✅ 正确：(\d{1,2})x(\d{1,2})

检查清单：

  • 是否有 2 个 ()？
  • 特殊字符是否转义？（. → \., ( → \(, [ → \[）
  • 重复范围是否用逗号？（{1,2} 不是 {1-2}）

Q: 同一个文件测试成功，但分析时失败

A: 规则可能被禁用了。

检查左侧"自定义规则"列表，确保 ☑️ 被勾选。

重命名相关

Q: 我担心文件会被破坏

A: 不用担心！有 3 层保护：

保护 1：预览功能

执行重命名前，总会显示预览结果。
你可以看到新文件名，确认无误後再执行。

保护 2：干运行模式（dry-run）

配置文件中设置 "behavior.dry_run": true
程序会显示结果但不实际重命名。

保护 3：自动备份

重命名前，原文件会自动备份。
即使出错也能恢复。

Q: 能否处理多个文件？

A: 支持两种方式：

方式 1：逐个处理（安全，推荐）

对每个文件：输入 → 分析 → 预览 → 执行
优点：可以逐个检查，安全可靠

方式 2：批量处理

配置中启用 "behavior.batch_mode": true
输入文件夹路径 → 程序自动处理所有匹配的文件
优点：速度快，适合大量文件

配置参考

配置文件位置 

./config/config.conf

配置文件结构 

{
  "api": {
    "key": "你的 TMDB API 密钥（首次自动创建）",
    "language": "zh-CN（中文）或 en-US（英文）",
    "timeout": 10（API 超时时间秒数）
  },
  
  "naming": {
    "include_episode_name": true（是否包含集名称）,
    "episode_separator": "-"（集号前的分隔符）,
    "format_style": "standard"（输出格式风格）
  },
  
  "behavior": {
    "dry_run": false（true = 仅显示结果不执行）,
    "interactive": true（真实交互模式）,
    "batch_mode": false（false = 逐个处理，true = 批量处理）
  },
  
  "rules": {
    "custom": [
      {
        "name": "规则名称",
        "pattern": "正则表达式",
        "description": "描述",
        "enabled": true
      }
    ]
  }
}

配置参数详解

API 部分 api

参数 说明 示例值
key TMDB API 密钥 abc123def456...
language API 返回数据的语言代码 zh-CN 或 en-US
timeout API 请求超时时间（秒） 10

命名部分 naming

参数 说明 可选值
include_episode_name 是否在重命名中包含集的名称 true / false
episode_separator 集号前的分隔符 - / / _ 等
format_style 输出格式 standard 或自定义

行为部分 behavior

参数 说明 用途
dry_run 干运行模式，仅显示结果不实际操作 调试时设为 true
interactive 是否进行交互式确认 通常保持 true
batch_mode 批量处理模式 true = 一次处理多个文件

自定义规则 rules.custom

这是一个数组，存储用户创建的所有规则。每条规则包含：

  • name：规则名称
  • pattern：正则表达式
  • description：描述（可选）
  • enabled：是否启用

配置修改建议

场景 1：测试（不想损坏原文件） 

"behavior": {
  "dry_run": true,      // 只显示结果
  "interactive": true   // 每步都确认
}

场景 2：大批量处理 

"behavior": {
  "dry_run": false,
  "batch_mode": true,      // 启用批量模式
  "interactive": false     // 无需交互，自动处理
}

场景 3：获取英文电视剧信息 

"api": {
  "language": "en-US"
}

常用正则表达式速查

复制即用！记住：每个正则必须有 2 个 ()

标准格式库

1. 01x01 格式 

(\d{1,2})x(\d{1,2})

匹配： 01x01 1x5 12x99 等

2. S01E01 格式 

[Ss](\d{1,2})[Ee](\d{1,2})

匹配： S01E01 s01e01 S5E10 等

3. [01] 01 或 (01) 01 

[\[\(]\s*(\d{1,2})\s*[\]\)]\s*(\d{1,2})

匹配： [01] 01 (01) 01 [01]01 等

4. 01.01 格式 

(\d{1,2})\.(\d{1,2})

匹配： 01.01 1.5 12.99 等

5. 01-01 格式 

(\d{1,2})-(\d{1,2})

匹配： 01-01 1-5 12-99 等

6. 单纯数字：01 01（含空格） 

(\d{1,2})\s+(\d{1,2})

匹配： 01 01 1 5 12 99 等

7. 单纯数字：0101（无分隔） 

(\d{2})(\d{2})

匹配： 0101 0505 1299 等
注意： 只能匹配固定 4 位数字

8. 动画格式：Ep 01 

[Ee]p(?:isode)?\s*(\d{1,2})(?:[xX.-](\d{1,2}))?

匹配： Ep 01 Episode 01 EP01 等

9. 第 01 集格式（中文） 

第\s*(\d{1,2})\s*季\s*第?\s*(\d{1,2})

匹配： 第01季第01集 第1季第5集 等

10. 复杂混合格式 

[Ss]?(\d{1,2})[Xx-]([Ee]?(\d{1,2}))?

匹配： S01E01 01x01 s01-05 等

复制粘贴的完整列表 

01x01:                  (\d{1,2})x(\d{1,2})
S01E01:                 [Ss](\d{1,2})[Ee](\d{1,2})
[01] 01:                [\[\(]\s*(\d{1,2})\s*[\]\)]\s*(\d{1,2})
01.01:                  (\d{1,2})\.(\d{1,2})
01-01:                  (\d{1,2})-(\d{1,2})
01 01:                  (\d{1,2})\s+(\d{1,2})
0101:                   (\d{2})(\d{2})
Ep 01:                  [Ee]p(?:isode)?\s*(\d{1,2})
第01集（中文）:         第\s*(\d{1,2})\s*季\s*第?\s*(\d{1,2})

总结对比表

格式 规则 使用场景
01x01 (\d{1,2})x(\d{1,2}) 常见 BT / 下载工具
S01E01 [Ss](\d{1,2})[Ee](\d{1,2}) 标准西方电视格式
[01] 01 [\[\(]\s*(\d{1,2})\s*[\]\)]\s*(\d{1,2}) 日本动画
01.01 (\d{1,2})\.(\d{1,2}) 某些东欧网站
01-01 (\d{1,2})-(\d{1,2}) 某些下载工具

最佳实践

✅ 推荐做法

  1. 简单可靠的规则优于复杂规则

    好：(\d{1,2})x(\d{1,2})
不好：[\d]{1,2}[xX]{1}[\d]{1,2}(太复杂)

  2. 为相似的格式声明多条规则

    格式：S01E01 和 s01e01
做法：创建一条 [Ss](\d{1,2})[Ee](\d{1,2})
     而不是 (\d{1,2})x(\d{1,2}) 和 ...

  3. 给规则取清晰的名字

    好：S##E##标准格式、国内BT网格式、日本动画格式
不好：规则1、规则2、regex123

❌ 容易出错

  1. ❌ 忘记捕获组：\d{1,2}x\d{1,2} → 会失败！
  2. ❌ 3 个或更多捕获组 → 多的会被忽略
  3. ❌ 特殊字符未转义：. 应写成 \.
  4. ❌ 过度复杂化规则 → 难以维护
