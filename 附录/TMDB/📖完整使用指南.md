# Phase 2 规则系统完全指南

## 什么是 Phase 2 规则系统？

**Phase 2 规则 = 模板系统 + 替换规则**

Phase 2 规则是一个现代化的、灵活的文件名重命名规则系统。与传统的正则表达式匹配相比，Phase 2 采用**模板系统**（sourceTemplate → targetTemplate）来定义规则，更直观易用。

### 核心特性

- ✅ **模板化定义** - 使用 `${变量}` 语法，一目了然
- ✅ **智能提取** - 自动从文件名中提取需要的信息（季号、集号、剧名等）
- ✅ **灵活生成** - 按任意格式重新组织文件名
- ✅ **分类管理** - 按电视剧或场景分类组织规则
- ✅ **实时预览** - 修改时立即看到效果
- ✅ **一键应用** - 保存规则后永久复用

## 规则分类

Phase 2 规则按**类别**组织：

### 1. 全局类别（通用规则）
```
├─ 常规
│  ├─ 标准 01x01 格式
│  ├─ 标准 S##E## 格式
│  └─ 其他通用格式
```

### 2. 电视剧类别（针对特定剧）
```
├─ Breaking Bad
│  ├─ Breaking Bad - 点号分隔格式
│  └─ Breaking Bad - 括号格式
├─ 权力游戏
│  ├─ 权力游戏 - 标准格式
│  └─ 权力游戏 - 特殊格式
```

当你在弹框中选定某个电视剧时，系统会**自动启用**该剧对应的规则。

## 规则的结构

```json
{
  "id": "rule_20260301_001",
  "name": "Breaking Bad - 标准格式",
  "description": "Breaking Bad 常用的重命名格式",
  "sourceTemplate": "${title} ${S}x${E} ${name}.${ext}",
  "targetTemplate": "${title} - S${S}E${E} - ${name}.${ext}",
  "extractionStrategy": "numeric",
  "enabled": true,
  "createdAt": "2026-02-28T10:00:00Z",
  "modifiedAt": "2026-02-28T14:30:00Z"
}
```

### 字段说明

| 字段 | 说明 | 示例 |
|------|------|------|
| `id` | 规则的唯一标识 | `rule_20260301_001` |
| `name` | 规则的显示名称 | `Breaking Bad - 标准格式` |
| `description` | 规则描述（可选） | `Breaking Bad 常用的重命名格式` |
| `sourceTemplate` | **源模板**（原文件名格式） | `${title} ${S}x${E} ${name}.${ext}` |
| `targetTemplate` | **目标模板**（新文件名格式） | `${title} - S${S}E${E} - ${name}.${ext}` |
| `extractionStrategy` | 提取策略 | `numeric` \| `position` \| `manual` |
| `enabled` | 是否启用 | `true` \| `false` |
| `createdAt` | 创建时间 | ISO 8601 格式 |
| `modifiedAt` | 最后修改时间 | ISO 8601 格式 |

## 模板系统详解

### 支持的变量

系统内置以下标准变量，可在源模板和目标模板中使用：

| 变量 | 默认别名 | 说明 | 示例 |
|------|--------|------|------|
| 季号 | `${S}` | 电视剧/剧集的季数 | `1`, `02`, `第1季` |
| 集号 | `${E}` | 电视剧/剧集的集数 | `1`, `08`, `第8集` |
| 电视剧名 | `${title}` | 电视剧的名称 | `Breaking Bad` |
| 集名 | `${name}` | 每集的名称 | `Pilot`, `Salud` |
| 年份 | `${year}` | 首映或发布年份 | `2008`, `2009` |
| 扩展名 | `${ext}` | 文件扩展名 | `mkv`, `mp4`, `avi` |
| 前缀 | `${prefix}` | 自定义前缀 | `[字幕组]`, `[HD]` |
| 后缀 | `${suffix}` | 自定义后缀 | `[完]`, `[END]` |

### 模板示例

#### 示例 1：简单的 01x01 格式转换

```
原文件名：    Breaking Bad 01x01 Pilot.mkv
              ↓
源模板：      ${title} ${S}x${E} ${name}.${ext}
              ↓
提取结果：    title=Breaking Bad, S=01, E=01, name=Pilot, ext=mkv
              ↓
目标模板：    ${title} - S${S}E${E} - ${name}.${ext}
              ↓
新文件名：    Breaking Bad - S01E01 - Pilot.mkv
```

#### 示例 2：带年份和前缀

```
原文件名：    [字幕组] Breaking Bad S01E08 Salud (2009).mkv
              ↓
源模板：      ${prefix} ${title} S${S}E${E} ${name} (${year}).${ext}
              ↓
提取结果：    prefix=[字幕组], title=Breaking Bad, S=01, E=08,
              name=Salud, year=2009, ext=mkv
              ↓
目标模板：    ${title} - ${year} - S${S}E${E} - ${name}.${ext}
              ↓
新文件名：    Breaking Bad - 2009 - S01E08 - Salud.mkv
```

#### 示例 3：点号格式转换

```
原文件名：    权力游戏 2.08.mkv
              ↓
源模板：      ${title} ${S}.${E}.${ext}
              ↓
提取结果：    title=权力游戏, S=2, E=08, ext=mkv
              ↓
目标模板：    ${title} - S${S}E${E}.${ext}
              ↓
新文件名：    权力游戏 - S2E8.mkv
```

## 如何在 UI 中创建规则

### 步骤 1：打开 Phase 2 规则弹框

在 Web 界面左侧，点击"Phase 2 规则"部分的任意选项，打开规则管理弹框。

### 步骤 2：选择或创建类别

**创建新类别（可选）：**
1. 在弹框上方的"分类名称"输入框输入（如 `Breaking Bad`）
2. 点"添加分类"
3. 新类别会出现在左侧列表

### 步骤 3：输入源模板

**源模板 = 你现有的文件名格式**

1. 点"添加规则"按钮
2. 在"源模板"输入框输入，使用 `${变量}` 语法
3. 示例：`${title} ${S}x${E} ${name}.${ext}`

**编写源模板的技巧：**
- 看你的实际文件名，逐个替换为对应的 `${变量}`
- 不需要的部分可以省略（如如果文件名中没有年份，就不要在源模板中写 `${year}`）
- 使用 `${prefix}` 和 `${suffix}` 来匹配前缀和后缀

### 步骤 4：输入目标模板

**目标模板 = 你希望的新文件名格式**

1. 在"目标模板"输入框输入，使用 `${变量}` 语法
2. 示例：`${title} - S${S}E${E} - ${name}.${ext}`

### 步骤 5：（可选）选择提取策略

系统支持三种提取策略：

| 策略 | 说明 | 适用场景 |
|------|------|--------|
| `numeric` | 优先识别数值，按大小排序 | **推荐** - 数值清晰 |
| `position` | 按模板位置从左到右提取 | 格式规范固定 |
| `manual` | 用户手动指定每个变量 | 复杂非标准格式 |

**默认使用 `numeric` 策略。**

### 步骤 6：查看实时预览

1. 在弹框下方"测试文件名"输入框输入一个实际的文件名（如 `Breaking Bad 01x01 Pilot.mkv`）
2. 点"预览"
3. 系统立即显示：
   - ✅ **提取结果** - 从源模板中识别出的变量值
   - ✅ **新文件名** - 按目标模板生成的结果

### 步骤 7：保存规则

1. 在"规则名称"输入框输入规则的显示名称（如 `Breaking Bad - 标准格式`）
2. （可选）在"规则描述"输入框添加描述
3. 点"保存规则"
4. 规则立即保存，会出现在左侧列表中

# 常见问题 & 故障排除

## 启动和配置

### Q: 启动时报错 "ModuleNotFoundError"

**A:** 安装缺失的依赖库
```bash
pip install -r requirements-tmdb.txt
```

或手动安装：
```bash
pip install requests
```

### Q: 访问 localhost:8000 打不开

**A:** 检查以下几点：
1. 程序是否正确启动了？（看执行窗口有没有错误）
2. 是否在输入地址时用了正确的端口？
3. 防火墙是否阻止了连接？

### Q: API 密钥无效

**A:** 检查：
- [ ] 密钥是否从官方来源复制：https://www.themoviedb.org/settings/api
- [ ] 是否有多余空格？
- [ ] 网络连接是否正常？
- [ ] 密钥是否过期？（在 TMDB 重新生成）

## 文件名识别

### Q: 分析文件名时显示"格式未识别"

**A:** 有两个原因，按顺序排查：

**原因 1：你的格式不在内置规则中**
```
解决：创建自定义规则
  1. 在"自定义正则表达式"框输入你的格式对应的正则
  2. 点"测试正则"验证
  3. 点"保存为规则"永久保存
  详见：【自定义规则完全指南】
```

**原因 2：你的规则被禁用了**
```
解决：启用规则
  1. 查看左侧"自定义规则"列表
  2. 找到对应的规则
  3. 勾选 ☑️ 启用它
```

### Q: 正则表达式测试失败

**A:** 99% 的原因是这 3 个问题：

**问题 1：缺少捕获组 `()`**
```
❌ 错误：\d{1,2}x\d{1,2}
✅ 正确：(\d{1,2})x(\d{1,2})
```

**问题 2：特殊符号未转义**
```
❌ 错误：(\d.2)x(\d.2)        点号没转义！
✅ 正确：(\d{1,2})x(\d{1,2})
```

**问题 3：重复范围错误**
```
❌ 错误：(\d{1-2})x(\d{1-2})   用了 - 代替 ,
✅ 正确：(\d{1,2})x(\d{1,2})
```

**检查清单：**
- [ ] 是否有 2 个 `()`？
- [ ] 特殊字符是否转义？（`.` → `\.`, `(` → `\(`, `[` → `\[`）
- [ ] 重复范围是否用逗号？（`{1,2}` 不是 `{1-2}`）

### Q: 同一个文件测试成功，但分析时失败

**A:** 规则可能被禁用了。
```
检查左侧"自定义规则"列表，确保 ☑️ 被勾选。
```

```json
    "format_style": "standard"
  },
  
  "behavior": {
    "dry_run": false,
    "interactive": true,
    "batch_mode": false
  },
  
  "variables": {
    "电影名": "${title}",
    "季号": "${S}",
    "集号": "${E}",
    "年份": "${year}",
    "前缀": "${prefix}",
    "后缀": "${suffix}",
    "扩展名": "${ext}",
    "自定义1": "${custom1}"
  },
  
  "rules": {
    "custom": [
      {
```

## 配置参数详解

### API 部分 `api`

| 参数 | 说明 | 示例值 |
|------|------|--------|
| `key` | TMDB API 密钥 | `abc123def456...` |
| `language` | API 返回数据的语言代码 | `zh-CN` 或 `en-US` |
| `timeout` | API 请求超时时间（秒） | `10` |

### 命名部分 `naming`

| 参数 | 说明 | 可选值 |
|------|------|--------|
| `include_episode_name` | 是否在重命名中包含集的名称 | `true` / `false` |
| `episode_separator` | 集号前的分隔符 | `-` / ` ` / `_` 等 |
| `format_style` | 输出格式 | `standard` 或自定义 |

### 行为部分 `behavior`

| 参数 | 说明 | 用途 |
|------|------|------|
| `dry_run` | 干运行模式，仅显示结果不实际操作 | 调试时设为 `true` |
| `interactive` | 是否进行交互式确认 | 通常保持 `true` |
| `batch_mode` | 批量处理模式 | `true` = 一次处理多个文件 |

### 全局变量定义 `variables`

这个对象定义所有规则默认使用的**全局**变量别名映射。每个变量对应一个别名，规则可选择性覆盖。

**结构：**
```json
"variables": {
  "标准变量名": "${别名}",
  "电影名": "${title}",
  "季号": "${S}"
}
```

**说明：**
- **键**：标准变量名（不需要 `${}` 包装）
- **值**：该变量在模板中使用的别名（需要 `${}` 包装）
- **全局生效**：对所有规则默认生效
- **可被覆盖**：规则可在自己的 `variables` 中重新定义某些变量

**预设全局变量（可扩展）：**
- `电影名`：电视剧/电影名称
- `季号`：季数
- `集号`：集数
- `年份`：首映或发布年份
- `前缀`：自定义前缀
- `后缀`：自定义后缀
- `扩展名`：文件扩展名
- `自定义1`-`自定义n`：用户自定义变量

**全局配置示例：**
```json
"variables": {
  "电影名": "${title}",
  "季号": "${S}",
  "集号": "${E}",
  "年份": "${year}",
  "前缀": "${prefix}",
  "后缀": "${suffix}",
  "扩展名": "${ext}",
  "自定义1": "${custom1}"
}
```

**两层变量定义优先级：**
1. **规则级别 variables** - 该规则的别名覆盖（最高）
2. **全局 variables** - 所有规则的默认别名（次高）

### 自定义规则 `rules.custom`

这是一个数组，存储用户创建的所有规则。每条规则包含：
- `name`：规则名称
- `sourceTemplate`：源文件名模板
- `targetTemplate`：目标文件名模板
- `variables`：规则级别的变量别名覆盖（可选）
- `extractionStrategy`：提取策略，支持 `position`/`numeric`/`manual`
- `exampleInput` 和 `exampleOutput`：示例输入和输出
- `description`：规则描述（可选）
- `enabled`：是否启用
- `pattern`：正则表达式（仅供旧版兼容性）

**规则级别的 variables 说明：**

这是一个**简化的键值对对象**，用于在该规则中覆盖全局变量的别名定义。

**格式：**
```json
"variables": {
  "标准变量名": "${别名}",
  "标准变量名2": "${别名2}"
}
```

**特点：**
- **键**：标准变量名（不需要 `${}` 包装）
- **值**：单个别名（需要 `${}` 包装）
- **覆盖范围**：仅影响这个规则，不影响其他规则
- **继承性**：未在规则中定义的变量使用全局配置
- **优先级**：规则级别定义 > 全局定义

**规则级别变量示例：**
```json
{
  "name": "Breaking Bad 标准格式",
  "sourceTemplate": "${电影名} ${季号}.${集号} - ${集名} (${年份}).${扩展名}",
  "targetTemplate": "${电影名} - S${季号}E${集号} - ${集名} (${年份}).${扩展名}",
  "variables": {
    "季号": "${S}",
    "集号": "${E}",
    "电影名": "${title}",
    "年份": "${year}"
  },
  "extractionStrategy": "numeric",
  "exampleInput": "Breaking Bad 2.08 - Salud (2009).mkv",
  "exampleOutput": "Breaking Bad - S02E08 - Salud (2009).mkv",
  "enabled": true
}
```

**何时使用规则级别变量：**
- 该规则有特殊的变量命名习惯
- 需要限制某个变量的别名数量（提高性能）
- 该规则不需要全局配置中的所有别名
- 示例：简单脚本场景可能只需要 `${S}` 而不需要 `${season}`

**何时只用全局配置：**
- 所有规则都使用相同的变量别名
- 变量命名习惯统一
- 不想在每条规则中重复配置

# 常用正则表达式速查

**复制即用！记住：每个正则必须有 2 个 `()`**

## 标准格式库

### 1. 01x01 格式
```regex
(\d{1,2})x(\d{1,2})
```
**匹配：** `01x01` `1x5` `12x99` 等

### 2. S01E01 格式
```regex
[Ss](\d{1,2})[Ee](\d{1,2})
```
**匹配：** `S01E01` `s01e01` `S5E10` 等

### 3. [01] 01 或 (01) 01
```regex
[\[\(]\s*(\d{1,2})\s*[\]\)]\s*(\d{1,2})
```
**匹配：** `[01] 01` `(01) 01` `[01]01` 等

### 4. 01.01 格式
```regex
(\d{1,2})\.(\d{1,2})
```
**匹配：** `01.01` `1.5` `12.99` 等

### 5. 01-01 格式
```regex
(\d{1,2})-(\d{1,2})
```
**匹配：** `01-01` `1-5` `12-99` 等

### 6. 单纯数字：01 01（含空格）
```regex
(\d{1,2})\s+(\d{1,2})
```
**匹配：** `01 01` `1 5` `12  99` 等

### 7. 单纯数字：0101（无分隔）
```regex
(\d{2})(\d{2})
```
**匹配：** `0101` `0505` `1299` 等  
**注意：** 只能匹配固定 4 位数字

### 8. 动画格式：Ep 01
```regex
[Ee]p(?:isode)?\s*(\d{1,2})(?:[xX.-](\d{1,2}))?
```
**匹配：** `Ep 01` `Episode 01` `EP01` 等

### 9. 第 01 集格式（中文）
```regex
第\s*(\d{1,2})\s*季\s*第?\s*(\d{1,2})
```
**匹配：** `第01季第01集` `第1季第5集` 等

### 10. 复杂混合格式
```regex
[Ss]?(\d{1,2})[Xx-]([Ee]?(\d{1,2}))?
```
**匹配：** `S01E01` `01x01` `s01-05` 等

## 复制粘贴的完整列表

```
01x01:                  (\d{1,2})x(\d{1,2})
S01E01:                 [Ss](\d{1,2})[Ee](\d{1,2})
[01] 01:                [\[\(]\s*(\d{1,2})\s*[\]\)]\s*(\d{1,2})
01.01:                  (\d{1,2})\.(\d{1,2})
01-01:                  (\d{1,2})-(\d{1,2})
01 01:                  (\d{1,2})\s+(\d{1,2})
0101:                   (\d{2})(\d{2})
Ep 01:                  [Ee]p(?:isode)?\s*(\d{1,2})
第01集（中文）:         第\s*(\d{1,2})\s*季\s*第?\s*(\d{1,2})
```

## 最佳实践

### ✅ 推荐做法

1. **简单可靠的规则优于复杂规则**
   ```
   好：(\d{1,2})x(\d{1,2})
   不好：[\d]{1,2}[xX]{1}[\d]{1,2}(太复杂)
   ```

2. **为相似的格式声明多条规则**
   ```
   格式：S01E01 和 s01e01
   做法：创建一条 [Ss](\d{1,2})[Ee](\d{1,2})
        而不是 (\d{1,2})x(\d{1,2}) 和 ...
   ```

3. **给规则取清晰的名字**
   ```
   好：S##E##标准格式、国内BT网格式、日本动画格式
   不好：规则1、规则2、regex123
   ```

### ❌ 容易出错

1. ❌ 忘记捕获组：`\d{1,2}x\d{1,2}` → 会失败！
2. ❌ 3 个或更多捕获组 → 多的会被忽略
3. ❌ 特殊字符未转义：`.` 应写成 `\.`
4. ❌ 过度复杂化规则 → 难以维护

### 优化 3：规则结构扩展

**当前规则结构：**
```json
{
  "name": "规则名称",
  "pattern": "正则表达式",
  "description": "描述",
  "enabled": true
}
```

**新增字段：**
```json
{
  "name": "规则名称",
  "pattern": "正则表达式",
  "description": "描述",
  "enabled": true,
  "example_input": "原始文件名示例",
  "example_output": "改名后的文件名示例",
  "tags": ["BT下载", "美剧"],
  "created_date": "2024-01-01",
  "last_used": "2024-02-28",
  "usage_count": 42
}
```

**新字段说明：**
- `example_input`：用于规则说明的输入示例
- `example_output`：改名后的文件名示例
- `tags`：规则标签，便于分类和搜索
- `created_date`：规则创建日期
- `last_used`：最后使用时间
- `usage_count`：使用次数

**智能规则生成：**

用户可以通过以下方式快速创建规则：

```
方式 1：直接输入原始和改名后的文件名
  输入1：Breaking Bad 01x01 Pilot.mkv
  输入2：Breaking Bad - 1x01 - Pilot.mkv
  → 系统自动推断规则和其他参数

方式 2：通过规则编辑器手动创建
  1. 输入规则名称
  2. 编写正则表达式
  3. 输入测试文件名和预期输出
  4. 系统自动生成规则（包括 description 和其他元数据）
```

### 优化 4：用户端数据处理

**目标：** 减少服务器负担，提升响应速度

**迁移到用户端的功能：**

| 功能 | 当前 | 优化后 |
|------|------|--------|
| 正则表达式测试 | 服务器 | 用户端（JavaScript）|
| TMDB 搜索结果处理 | 服务器 | 用户端（数据结构化）|
| 规则预览生成 | 计划中 | 用户端（实时预览）|
| 规则信息填充 | 无 | 用户端（自动填充）|
| 文件名分析 | 计划中 | 用户端（快速反应）|

**JavaScript 增强功能列表：**

```javascript
// 1. 正则表达式测试
testRegex(pattern, testString)
  → {success: bool, matched: str, season: int, episode: int}

// 2. TMDB 数据处理

  → 构建化的数据结构

// 3. 规则预览
previewRuleMatch(pattern, testString, format)
  → 预览改名后的文件名

// 4. 规则智能生成

  → 推断的规则对象

// 5. 文件名分析

  → 识别季集号和适用规则
```

**创建规则：**

### 2. 模板系统的实现 - 核心设计1

**模板解析引擎**

系统需要实现一个轻量级的模板解析引擎，支持以下功能：

```javascript
// 0. 变量别名正规化
normalizeVariableNames(template, ruleVariables, globalVariables)
  // ${S} → 查询规则配置 → 确认为 ${季号}
  // 支持两层优先级：规则级别 > 全局定义
  // 返回标准化后的模板

// 1. 模板变量识别
parseTemplate("${前缀} - E${季号}S${集号} (${年份}).${扩展名}")
→ ['前缀', '季号', '集号', '年份', '扩展名']

// 2. 文件名智能提取
extractVariables(
  filename: "测试test.12第13（2025）.mkv",
  sourceTemplate: "${前缀}.${季号}${集号}（${年份}）.${扩展名}",
  extractionStrategy: "position | numeric",
  ruleVariables: {...},      // 规则级别的变量覆盖（可选）
  globalVariables: {...}     // 全局变量定义
)
→ { 前缀: "测试test", 季号: "12", 集号: "13", 年份: "2025", 扩展名: "mkv" }

// 3. 目标文件名生成
generateFilename(
  variables: { 前缀: "测试test", 季号: "12", 集号: "13", 年份: "2025", 扩展名: "mkv" },
  targetTemplate: "${前缀} - E${季号}S${集号} (${年份}).${扩展名}",
  ruleVariables: {...},      // 规则级别的变量覆盖（可选）
  globalVariables: {...}     // 全局变量定义
)
→ "测试test - E12S13 (2025).mkv"
```

**变量正规化算法：**
```
对于模板中的每个 ${aliasName}：
  1. 查询规则级别变量映射，是否存在？
     是 → 使用规则级别的定义，标准化为对应的标准变量名
     否 → 转到第 2 步
  
  2. 查询全局变量定义，该别名属于哪个标准变量？
     找到 → 标准化为该标准变量名
     未找到 → 报错（未定义的别名）
  
  3. 返回标准化的变量名（如 ${季号}）
```

**提取策略说明**

为了处理复杂的文件名格式，系统支持多种提取策略：

| 策略 | 说明 | 适用场景 |
|------|------|--------|
| `position` | 按模板位置顺序从左到右提取 | 格式规范，位置固定 |
| `numeric` | 优先提取数值，按大小排序 | 数值作为关键标识符 |
| `manual` | 用户手动指定每个变量的来源 | 复杂/非标准格式 |

### 2. 替换规则的实现 - 核心设计2

**替换规则的匹配和执行算法：**

```
当用户提交文件名和可选的电视剧/电影选项时：

第 1 步：加载可用的替换规则
  - 从配置中读取"常规"分类下的所有规则
  - 如果用户已选定电视剧，加载对应的"电视剧_XXX"规则
  - 如果用户已选定电影，加载对应的"电影_XXX"规则

第 2 步：规则匹配和应用（按优先级）
  - 优先尝试专用规则（电视剧/电影_XXX）
  - 如果失败，回退到"常规"规则
  - 对每条规则：
    a) 调用模板系统的 extractVariables() 尝试提取
    b) 如果提取成功，调用 generateFilename() 生成新名字
    c) 如果生成成功，返回预览结果
    d) 如果失败，继续下一条规则

第 3 步：结果处理
  - 匹配成功 → 显示预览，等待用户确认
  - 全部失败 → 提示"未找到匹配的规则"，建议用户创建新规则
```

**规则存储和加载：**

替换规则存储在配置文件中的 `substitutionRules` 节点下，格式如上面的例子所示。系统启动时加载所有规则，并维护一个规则索引表以支持快速查询。

**优化 4：用户端数据处理**

所有模板解析、变量提取和文件名生成都在用户端（JavaScript）进行：
- **模板语法验证** - 检测语法错误
- **变量映射处理** - 识别变量别名并转换为标准变量名
- **实时预览** - 用户输入时立即显示效果
- **性能优化** - 避免往返服务器
- **离线支持** - 规则编辑不依赖网络

**变量映射流程**：
```javascript
前置条件：
  全局配置中的变量定义：
    "季号": "${S}"
    "集号": "${E}"
    "电影名": "${title}"

第 1 层：全局变量定义
  定义所有变量名和它们在模板中的别名
  
第 2 层：规则级别覆盖（可选）
  该规则定义（如非必要可省略）：
    "季号": "${season}"        // 该规则使用 season 而不是 S
    "集号": "${ep}"            // 该规则使用 ep 而不是 E

第 3 层：模板解析
  源模板：  "${title} ${season}E${ep}"
  变量识别：${title}, ${season}, ${ep}
  
第 4 层：别名正规化
  ${season} → 查询规则变量 → 确认映射到 ${季号}
  ${ep}     → 查询规则变量 → 确认映射到 ${集号}
  ${title}  → 查询全局定义（规则未覆盖） → 映射到 ${电影名}

第 5 层：标准化后的内部模板
  "${电影名} ${季号}E${集号}"
```

**优先级链：**
```
规则级别变量 > 全局变量定义
```

**变量不匹配处理：**
- 如果模板中使用了未定义的别名，系统会报错
- 用户需要在全局配置或规则级别 variables 中添加该别名
