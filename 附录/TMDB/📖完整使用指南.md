# 自定义规则完全指南

## 什么是自定义规则？

**规则 = 正则表达式**

规则用来识别和提取文件名中的季号和集号。

## 规则的结构

```json
{
  "name": "规则的名称（如：01x01格式）",
  "pattern": "正则表达式（如：(\\d{1,2})x(\\d{1,2})）",
  "description": "规则描述（可选，如：标准的##x##格式）",
  "enabled": true 或 false（是否启用）
}
```

## 正则表达式基础速成

### 关键符号速查

| 符号 | 含义 | 示例 |
|------|------|------|
| `()` | **捕获组**（必须有 2 个！） | `(\d+)` |
| `\d` | 任意数字 [0-9] | `\d{1,2}` 匹配 1-2 位数字 |
| `{1,2}` | 重复次数 1 到 2 次 | `\d{1,2}` |
| `\s` | 空白字符（空格、制表符等） | `\s*` 匹配 0 个或多个空格 |
| `[...]` | 字符集 | `[a-zA-Z]` 匹配字母 |
| `\|` | 或者 | `S\|s` 匹配 S 或 s |
| `.` | 任意字符（除换行） | 通常需要转义 `\.` |
| `^` | 字符串开头 | `^S01` |
| `$` | 字符串末尾 | `E01$` |

### 必须记住：2 个捕获组

```
✅ 正确：   (\d{1,2})x(\d{1,2})
           第 1 组(季)  第 2 组(集)

❌ 错误：   \d{1,2}x\d{1,2}        → 没有括号！
❌ 错误：   (\d{1,2})               → 只有 1 个括号！
❌ 错误：   (\d{1,2})-(\d{1,2})-(\d{1,2}) → 3 个括号！
```

## 常见文件格式和规则

### 格式 1：01x01
```
文件名示例：Breaking Bad 01x01 Pilot.mkv
正则表达式：(\d{1,2})x(\d{1,2})
规则名称：01x01 格式
```

### 格式 2：S01E01
```
文件名示例：Breaking Bad S01E01 Pilot.mkv
正则表达式：[Ss](\d{1,2})[Ee](\d{1,2})
规则名称：S##E##标准格式
```

### 格式 3：[01] 01（动画格式）
```
文件名示例：[Anime] Title [01] 01 Title.mkv
正则表达式：[\[\(]\s*(\d{1,2})\s*[\]\)]\s*(\d{1,2})
规则名称：动画 [##] ## 格式
说明：支持 [01] 或 (01) 两种括号
```

### 格式 4：01.01
```
文件名示例：Breaking Bad 01.01 Pilot.mkv
正则表达式：(\d{1,2})\.(\d{1,2})
规则名称：点号格式
```

### 格式 5：01-01
```
文件名示例：Breaking Bad 01-01 Pilot.mkv
正则表达式：(\d{1,2})-(\d{1,2})
规则名称：连字号格式
```

## 文件名识别

### Q: 分析文件名时显示"格式未识别"

**A:** 有两个原因，按顺序排查：

**原因 1：你的格式不在内置规则中**
```
解决：创建自定义规则
  1. 在"自定义正则表达式"框输入你的格式对应的正则
  2. 点"测试正则"验证
  3. 点"保存为规则"永久保存
  详见：【自定义规则完全指南】
```

**原因 2：你的规则被禁用了**
```
解决：启用规则
  1. 查看左侧"自定义规则"列表
  2. 找到对应的规则
  3. 勾选 ☑️ 启用它
```

### Q: 正则表达式测试失败

**A:** 99% 的原因是这 3 个问题：

**问题 1：缺少捕获组 `()`**
```
❌ 错误：\d{1,2}x\d{1,2}
✅ 正确：(\d{1,2})x(\d{1,2})
```

**问题 2：特殊符号未转义**
```
❌ 错误：(\d.2)x(\d.2)        点号没转义！
✅ 正确：(\d{1,2})x(\d{1,2})
```

**问题 3：重复范围错误**
```
❌ 错误：(\d{1-2})x(\d{1-2})   用了 - 代替 ,
✅ 正确：(\d{1,2})x(\d{1,2})
```

**检查清单：**
- [ ] 是否有 2 个 `()`？
- [ ] 特殊字符是否转义？（`.` → `\.`, `(` → `\(`, `[` → `\[`）
- [ ] 重复范围是否用逗号？（`{1,2}` 不是 `{1-2}`）

### Q: 同一个文件测试成功，但分析时失败

**A:** 规则可能被禁用了。
```
检查左侧"自定义规则"列表，确保 ☑️ 被勾选。
```

## 重命名相关

### Q: 我担心文件会被破坏

**A:** 不用担心！有 3 层保护：

**保护 1：预览功能**
```
执行重命名前，总会显示预览结果。
你可以看到新文件名，确认无误後再执行。
```

**保护 2：干运行模式（dry-run）**
```
配置文件中设置 "behavior.dry_run": true
程序会显示结果但不实际重命名。
```

**保护 3：自动备份**
```
重命名前，原文件会自动备份。
即使出错也能恢复。
```

### Q: 能否处理多个文件？

**A:** 支持两种方式：

**方式 1：逐个处理（安全，推荐）**
```
对每个文件：输入 → 分析 → 预览 → 执行
优点：可以逐个检查，安全可靠
```

**方式 2：批量处理**
```
配置中启用 "behavior.batch_mode": true
输入文件夹路径 → 程序自动处理所有匹配的文件
优点：速度快，适合大量文件
```

# 配置参考

## 配置文件位置

```
./config/config.conf
```

## 配置文件结构

```json
{
  "api": {
    "key": "你的 TMDB API 密钥（首次自动创建）",
    "language": "zh-CN（中文）或 en-US（英文）",
    "timeout": 10
  },
  
  "naming": {
    "include_episode_name": true,
    "episode_separator": "-",
    "format_style": "standard"
  },
  
  "behavior": {
    "dry_run": false,
    "interactive": true,
    "batch_mode": false
  },
  
  "variables": {
    "电影名": "${title}",
    "季号": "${S}",
    "集号": "${E}",
    "年份": "${year}",
    "前缀": "${prefix}",
    "后缀": "${suffix}",
    "扩展名": "${ext}",
    "自定义1": "${custom1}"
  },
  
  "rules": {
    "custom": [
      {
        "name": "规则名称",
        "sourceTemplate": "${电影名} ${季号}.${集号}.${扩展名}",
        "targetTemplate": "${电影名} - S${季号}E${集号}.${扩展名}",
        "extractionStrategy": "position",
        "variables": {
          "季号": "${S}",
          "集号": "${E}"
        },
        "enabled": true
      }
    ]
  }
}
```

## 配置参数详解

### API 部分 `api`

| 参数 | 说明 | 示例值 |
|------|------|--------|
| `key` | TMDB API 密钥 | `abc123def456...` |
| `language` | API 返回数据的语言代码 | `zh-CN` 或 `en-US` |
| `timeout` | API 请求超时时间（秒） | `10` |

### 命名部分 `naming`

| 参数 | 说明 | 可选值 |
|------|------|--------|
| `include_episode_name` | 是否在重命名中包含集的名称 | `true` / `false` |
| `episode_separator` | 集号前的分隔符 | `-` / ` ` / `_` 等 |
| `format_style` | 输出格式 | `standard` 或自定义 |

### 行为部分 `behavior`

| 参数 | 说明 | 用途 |
|------|------|------|
| `dry_run` | 干运行模式，仅显示结果不实际操作 | 调试时设为 `true` |
| `interactive` | 是否进行交互式确认 | 通常保持 `true` |
| `batch_mode` | 批量处理模式 | `true` = 一次处理多个文件 |

### 全局变量定义 `variables`

这个对象定义所有规则默认使用的**全局**变量别名映射。每个变量对应一个别名，规则可选择性覆盖。

**结构：**
```json
"variables": {
  "标准变量名": "${别名}",
  "电影名": "${title}",
  "季号": "${S}"
}
```

**说明：**
- **键**：标准变量名（不需要 `${}` 包装）
- **值**：该变量在模板中使用的别名（需要 `${}` 包装）
- **全局生效**：对所有规则默认生效
- **可被覆盖**：规则可在自己的 `variables` 中重新定义某些变量

**预设全局变量（可扩展）：**
- `电影名`：电视剧/电影名称
- `季号`：季数
- `集号`：集数
- `年份`：首映或发布年份
- `前缀`：自定义前缀
- `后缀`：自定义后缀
- `扩展名`：文件扩展名
- `自定义1`-`自定义n`：用户自定义变量

**全局配置示例：**
```json
"variables": {
  "电影名": "${title}",
  "季号": "${S}",
  "集号": "${E}",
  "年份": "${year}",
  "前缀": "${prefix}",
  "后缀": "${suffix}",
  "扩展名": "${ext}",
  "自定义1": "${custom1}"
}
```

**两层变量定义优先级：**
1. **规则级别 variables** - 该规则的别名覆盖（最高）
2. **全局 variables** - 所有规则的默认别名（次高）

### 自定义规则 `rules.custom`

这是一个数组，存储用户创建的所有规则。每条规则包含：
- `name`：规则名称
- `sourceTemplate`：源文件名模板
- `targetTemplate`：目标文件名模板
- `variables`：规则级别的变量别名覆盖（可选）
- `extractionStrategy`：提取策略，支持 `position`/`numeric`/`manual`
- `exampleInput` 和 `exampleOutput`：示例输入和输出
- `description`：规则描述（可选）
- `enabled`：是否启用
- `pattern`：正则表达式（仅供旧版兼容性）

**规则级别的 variables 说明：**

这是一个**简化的键值对对象**，用于在该规则中覆盖全局变量的别名定义。

**格式：**
```json
"variables": {
  "标准变量名": "${别名}",
  "标准变量名2": "${别名2}"
}
```

**特点：**
- **键**：标准变量名（不需要 `${}` 包装）
- **值**：单个别名（需要 `${}` 包装）
- **覆盖范围**：仅影响这个规则，不影响其他规则
- **继承性**：未在规则中定义的变量使用全局配置
- **优先级**：规则级别定义 > 全局定义

**规则级别变量示例：**
```json
{
  "name": "Breaking Bad 标准格式",
  "sourceTemplate": "${电影名} ${季号}.${集号} - ${集名} (${年份}).${扩展名}",
  "targetTemplate": "${电影名} - S${季号}E${集号} - ${集名} (${年份}).${扩展名}",
  "variables": {
    "季号": "${S}",
    "集号": "${E}",
    "电影名": "${title}",
    "年份": "${year}"
  },
  "extractionStrategy": "numeric",
  "exampleInput": "Breaking Bad 2.08 - Salud (2009).mkv",
  "exampleOutput": "Breaking Bad - S02E08 - Salud (2009).mkv",
  "enabled": true
}
```

**何时使用规则级别变量：**
- 该规则有特殊的变量命名习惯
- 需要限制某个变量的别名数量（提高性能）
- 该规则不需要全局配置中的所有别名
- 示例：简单脚本场景可能只需要 `${S}` 而不需要 `${season}`

**何时只用全局配置：**
- 所有规则都使用相同的变量别名
- 变量命名习惯统一
- 不想在每条规则中重复配置

# 常用正则表达式速查

**复制即用！记住：每个正则必须有 2 个 `()`**

## 标准格式库

### 1. 01x01 格式
```regex
(\d{1,2})x(\d{1,2})
```
**匹配：** `01x01` `1x5` `12x99` 等

### 2. S01E01 格式
```regex
[Ss](\d{1,2})[Ee](\d{1,2})
```
**匹配：** `S01E01` `s01e01` `S5E10` 等

### 3. [01] 01 或 (01) 01
```regex
[\[\(]\s*(\d{1,2})\s*[\]\)]\s*(\d{1,2})
```
**匹配：** `[01] 01` `(01) 01` `[01]01` 等

### 4. 01.01 格式
```regex
(\d{1,2})\.(\d{1,2})
```
**匹配：** `01.01` `1.5` `12.99` 等

### 5. 01-01 格式
```regex
(\d{1,2})-(\d{1,2})
```
**匹配：** `01-01` `1-5` `12-99` 等

### 6. 单纯数字：01 01（含空格）
```regex
(\d{1,2})\s+(\d{1,2})
```
**匹配：** `01 01` `1 5` `12  99` 等

### 7. 单纯数字：0101（无分隔）
```regex
(\d{2})(\d{2})
```
**匹配：** `0101` `0505` `1299` 等  
**注意：** 只能匹配固定 4 位数字

### 8. 动画格式：Ep 01
```regex
[Ee]p(?:isode)?\s*(\d{1,2})(?:[xX.-](\d{1,2}))?
```
**匹配：** `Ep 01` `Episode 01` `EP01` 等

### 9. 第 01 集格式（中文）
```regex
第\s*(\d{1,2})\s*季\s*第?\s*(\d{1,2})
```
**匹配：** `第01季第01集` `第1季第5集` 等

### 10. 复杂混合格式
```regex
[Ss]?(\d{1,2})[Xx-]([Ee]?(\d{1,2}))?
```
**匹配：** `S01E01` `01x01` `s01-05` 等

## 复制粘贴的完整列表

```
01x01:                  (\d{1,2})x(\d{1,2})
S01E01:                 [Ss](\d{1,2})[Ee](\d{1,2})
[01] 01:                [\[\(]\s*(\d{1,2})\s*[\]\)]\s*(\d{1,2})
01.01:                  (\d{1,2})\.(\d{1,2})
01-01:                  (\d{1,2})-(\d{1,2})
01 01:                  (\d{1,2})\s+(\d{1,2})
0101:                   (\d{2})(\d{2})
Ep 01:                  [Ee]p(?:isode)?\s*(\d{1,2})
第01集（中文）:         第\s*(\d{1,2})\s*季\s*第?\s*(\d{1,2})
```

| 格式 | 规则 | 使用场景 |
|------|------|---------|
| `01x01` | `(\d{1,2})x(\d{1,2})` | 常见 BT / 下载工具 |
| `S01E01` | `[Ss](\d{1,2})[Ee](\d{1,2})` | 标准西方电视格式 |
| `[01] 01` | `[\[\(]\s*(\d{1,2})\s*[\]\)]\s*(\d{1,2})` | 日本动画 |
| `01.01` | `(\d{1,2})\.(\d{1,2})` | 某些东欧网站 |

## 最佳实践

### ✅ 推荐做法

1. **简单可靠的规则优于复杂规则**
   ```
   好：(\d{1,2})x(\d{1,2})
   不好：[\d]{1,2}[xX]{1}[\d]{1,2}(太复杂)
   ```

2. **为相似的格式声明多条规则**
   ```
   格式：S01E01 和 s01e01
   做法：创建一条 [Ss](\d{1,2})[Ee](\d{1,2})
        而不是 (\d{1,2})x(\d{1,2}) 和 ...
   ```

3. **给规则取清晰的名字**
   ```
   好：S##E##标准格式、国内BT网格式、日本动画格式
   不好：规则1、规则2、regex123
   ```

### ❌ 容易出错

1. ❌ 忘记捕获组：`\d{1,2}x\d{1,2}` → 会失败！
2. ❌ 3 个或更多捕获组 → 多的会被忽略
3. ❌ 特殊字符未转义：`.` 应写成 `\.`
4. ❌ 过度复杂化规则 → 难以维护

## 核心设计1：模板系统

用类似 Shell 变量替换的语法定义**源文件名模板**和**目标文件名模板**。

**预设变量（默认别名）：**

系统提供以下标准变量和它们在全局配置中的默认别名：

| 标准变量名 | 默认别名 | 说明 |
|----------|--------|---------|
| `电影名` | `${title}` | 电视剧/电影名称 |
| `季号` | `${S}` | 季数 |
| `集号` | `${E}` | 集数 |
| `年份` | `${year}` | 首映或发布年份 |
| `前缀` | `${prefix}` | 自定义前缀 |
| `后缀` | `${suffix}` | 自定义后缀 |
| `扩展名` | `${ext}` | 文件扩展名 |
| `自定义1`-`自定义n` | `${custom1}` 等 | 用户自定义变量 |

**别名说明：**
- 每个变量有一个**默认别名**（在全局 `variables` 配置中保存）
- 规则可以选择性地覆盖某些变量的别名
- 模板中使用别名（如 `${title}`）而非标准名（如 `${电影名}`）

**全局配置示例：**
```json
"variables": {
  "电影名": "${title}",
  "季号": "${S}",
  "集号": "${E}",
  "年份": "${year}",
  "前缀": "${prefix}",
  "后缀": "${suffix}",
  "扩展名": "${ext}",
  "自定义1": "${custom1}"
}
```

**示例 1：简单格式**
```
原文件名：  测试test.12第13.mkv
源模板：    ${集号}.${扩展名}
提取结果：  集号=13, 扩展名=mkv
           （前缀=空，季号=空，因为源模板中未定义）

目标模板：  ${前缀}E${季号}S${集号}.${扩展名}
生成文件名：E00S13.mkv
           （前缀为空，季号为0，集号=13）

⚠️ 说明：源模板中未定义的变量无法提取，即使文件名中包含相关信息也不用。
         要提取前缀和季号，需要在源模板中显式定义。
```

**示例 2：带年份的复杂格式**
```
原文件名：  测试test.12第13（2025）.mkv
源模板：    ${前缀}.${季号}${集号}（${年份}）.${扩展名}
提取结果：  前缀=测试test, 季号=12, 集号=13, 年份=2025

目标模板：  ${前缀} - E${季号}S${集号} (${年份}).${扩展名}
生成文件名：测试test - E12S13 (2025).mkv
```

**示例 3：使用自定义变量别名**
```
全局配置中定义的变量别名：
  "季号": "${S}"
  "集号": "${E}"

原文件名：  Series S02E05 Title.mkv
源模板：    ${title} ${S}${E} ${name}.${ext}
（使用别名 ${S} 和 ${E}，映射到 ${季号} 和 ${集号}）

提取结果：  title=Series, 季号=02, 集号=05, name=Title, ext=mkv
           （${S} 被识别为 ${季号}，${E} 被识别为 ${集号}）

目标模板：  ${title} - S${季号}E${集号} - ${name}.${ext}
生成文件名：Series - S02E05 - Title.mkv
```

**工作流程：**
```
1. 用户输入源模板和目标模板
2. 系统分析源模板，识别出需要从文件名中提取的变量（只处理源模板中显式定义的变量）
3. 根据文件名和源模板进行智能匹配和提取（未在源模板中定义的变量不提取）
4. 将提取的值代入目标模板生成新文件名（未提取的变量以空值/0替代）
5. 显示预览结果
```

**⚠️ 核心原则：**
- 只从源模板中**显式定义**的 `${变量}` 中提取值
- 目标模板中**引用的变量**如果在源模板未定义，则以空值（字符串）或0（数值）代替
- 这确保了提取的准确性和避免误判

**提取策略：**
- 从左到右遍历文件名，按照模板位置抽取数值分组
- 支持**位置优先**和**数值优先**两种策略
- 可配置**数值分组的匹配顺序**（首个、最后一个、手动指定）

## Phase 2 实现细节

### 1. 模板系统的实现 - 核心设计1

**模板解析引擎**

系统需要实现一个轻量级的模板解析引擎，支持以下功能：

```javascript
// 0. 变量别名正规化
normalizeVariableNames(template, ruleVariables, globalVariables)
  // ${S} → 查询规则配置 → 确认为 ${季号}
  // 支持两层优先级：规则级别 > 全局定义
  // 返回标准化后的模板

// 1. 模板变量识别
parseTemplate("${前缀} - E${季号}S${集号} (${年份}).${扩展名}")
→ ['前缀', '季号', '集号', '年份', '扩展名']

// 2. 文件名智能提取
extractVariables(
  filename: "测试test.12第13（2025）.mkv",
  sourceTemplate: "${前缀}.${季号}${集号}（${年份}）.${扩展名}",
  extractionStrategy: "position | numeric",
  ruleVariables: {...},      // 规则级别的变量覆盖（可选）
  globalVariables: {...}     // 全局变量定义
)
→ { 前缀: "测试test", 季号: "12", 集号: "13", 年份: "2025", 扩展名: "mkv" }

// 3. 目标文件名生成
generateFilename(
  variables: { 前缀: "测试test", 季号: "12", 集号: "13", 年份: "2025", 扩展名: "mkv" },
  targetTemplate: "${前缀} - E${季号}S${集号} (${年份}).${扩展名}",
  ruleVariables: {...},      // 规则级别的变量覆盖（可选）
  globalVariables: {...}     // 全局变量定义
)
→ "测试test - E12S13 (2025).mkv"
```

**变量正规化算法：**
```
对于模板中的每个 ${aliasName}：
  1. 查询规则级别变量映射，是否存在？
     是 → 使用规则级别的定义，标准化为对应的标准变量名
     否 → 转到第 2 步
  
  2. 查询全局变量定义，该别名属于哪个标准变量？
     找到 → 标准化为该标准变量名
     未找到 → 报错（未定义的别名）
  
  3. 返回标准化的变量名（如 ${季号}）
```

**提取策略说明**

为了处理复杂的文件名格式，系统支持多种提取策略：

| 策略 | 说明 | 适用场景 |
|------|------|--------|
| `position` | 按模板位置顺序从左到右提取 | 格式规范，位置固定 |
| `numeric` | 优先提取数值，按大小排序 | 数值作为关键标识符 |
| `manual` | 用户手动指定每个变量的来源 | 复杂/非标准格式 |

### 2. 替换规则的实现 - 核心设计2

**替换规则的匹配和执行算法：**

```
当用户提交文件名和可选的电视剧/电影选项时：

第 1 步：加载可用的替换规则
  - 从配置中读取"常规"分类下的所有规则
  - 如果用户已选定电视剧，加载对应的"电视剧_XXX"规则
  - 如果用户已选定电影，加载对应的"电影_XXX"规则

第 2 步：规则匹配和应用（按优先级）
  - 优先尝试专用规则（电视剧/电影_XXX）
  - 如果失败，回退到"常规"规则
  - 对每条规则：
    a) 调用模板系统的 extractVariables() 尝试提取
    b) 如果提取成功，调用 generateFilename() 生成新名字
    c) 如果生成成功，返回预览结果
    d) 如果失败，继续下一条规则

第 3 步：结果处理
  - 匹配成功 → 显示预览，等待用户确认
  - 全部失败 → 提示"未找到匹配的规则"，建议用户创建新规则
```

```javascript
  变量识别：${title}, ${season}, ${ep}
  
第 4 层：别名正规化
  ${season} → 查询规则变量 → 确认映射到 ${季号}
  ${ep}     → 查询规则变量 → 确认映射到 ${集号}
  ${title}  → 查询全局定义（规则未覆盖） → 映射到 ${电影名}

第 5 层：标准化后的内部模板
  "${电影名} ${季号}E${集号}"
```

**优先级链：**
```
规则级别变量 > 全局变量定义
```

**变量不匹配处理：**
- 如果模板中使用了未定义的别名，系统会报错
- 用户需要在全局配置或规则级别 variables 中添加该别名
